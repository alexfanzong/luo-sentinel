LUO SENTINEL · RWA ON INJECTIVE · AGENT × HUMAN COMPLIANCE LAYER

让 Agent 把准备做到位，让人守住最后一关。

LUO Sentinel 是 Agent 和链上执行之间的一道合规夹层。Agent 把 RWA 动作对齐到已审核的监管证据、整理出可执行的准备工作；人用钱包做最后确认；每一步都能追溯到具体来源。

prepareAgent 把分散的监管来源，整理成一份可执行的准备清单。
verify每个结论都落到能点开的来源，分歧也保留在记录里。
confirm人用钱包签下最后确认，留下一条可追溯的负责记录。

01 · 问题

Agent 很能干，但有些判断必须留给人。

RWA 项目会让 Agent 准备发行、转让和策略。Agent 擅长补全路径，也擅长把“看起来合理”的片段串成计划——这正是它的价值。但跨境监管常常没有唯一答案，更多是来源边界、分类问题，和要靠人来拍板的地方。

要补的，是一道把 Agent 的速度和人的判断都留得住的合规夹层。

分工

Agent 负责把准备做到位，要不要执行，交给人来定。

02 · 产品定义

Sentinel 是 Agent 和链上执行之间的一道合规夹层。

它不下“可以发行”的结论。它做的是更扎实的一步：把动作对齐到已审核来源，让 reviewer Agents 把范围、来源、风险都准备清楚，再交给人确认。

人用钱包确认后，系统生成 receipt 和 handoff。这个 handoff 只授权下游 Agent 在批准的边界内，准备给律师看的材料。

03 · Guarded handoff chain

一条让 Agent 和人各司其职的链路。

01 upstream

Agent 发起 RWA 动作

例如准备 OUSG-like tokenized treasury 在 Injective 上发行或转让。

02 sentinel

进入合规夹层

动作先对齐证据，在人确认前不上链，给后面每一步留出可核查的空间。

03 evidence

对齐已审核来源

映射到 US、HK、SG、EU 四个已审核信号，只在有据可依的范围内推进。

04 human gate

人来把最后一关

人看过证据后决定 Proceed 或 Hold，并用钱包签字负责。

05 downstream

下游 Agent 在边界内继续

拿到批准后，下游 Agent 在授权范围内生成律师准备清单。

05 · Reviewer Agents

三个 reviewer Agents 把范围、来源、风险准备清楚，交给人来判断。

scope agent

范围有没有覆盖到位

确认请求落在已审核的辖区和来源之内。

70
source agent

每个结论是否都有出处

确认每个 claim 都能点回来源，而不是模型自己补的。

72
risk agent

哪些动作需要人来确认

标出发行、转让、策略这些该由人和律师决定的环节。

68

设计边界

这个分是给人参考的审核权重，帮你更快看清重点；它不是模型置信度，也不是合规分数。

06 · Receipt protocol

上链的只有一句话：谁、在什么边界下、点了同意。

receiptHash: 0x...
evidenceHash: reviewed OUSG pack
productRefHash: OUSG-like treasury sample
reviewerWallet: 0x...
decision: proceed allowedDownstreamAction: counsel-prep checklist
forbidden: issuance, transfer, order, strategy, asset movement
network: Injective EVM testnet
value: 0 INJ

这份 receipt 是一条可追溯的负责记录：人在特定证据范围下，确认了一次有边界的 AI 协作。它不证明合规，不构成法律意见，也不授权资产交易。

07 · Demo path

Demo 想让你看见：Agent 和人怎么把一次 RWA 动作安全地走完。

主线 · 跨境 review

  1. 01Agent 发起跨境请求，进入合规夹层。
  2. 02地图打开四个已审核的来源信号。
  3. 03三个 reviewer Agents 准备好范围、来源、风险。
  4. 04人确认 Proceed，并用测试网钱包签字。
  5. 05下游 Agent 在边界内生成 counsel-prep checklist。

保护分支 · 超出已审核范围

  1. 01请求超出已审核来源范围。
  2. 02Sentinel 不编地图、不猜辖区，把你带回有据可依的范围。
  3. 03回到跨境范围或单一已审核来源继续。
  4. 04receipt 始终绑定被选择的来源范围。
  5. 05handoff 不会扩大到未审核地区。
