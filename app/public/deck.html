LUO SENTINEL · RWA ON INJECTIVE · AGENT GUARD LAYER

Agent 不该自己说服自己去发资产。

LUO Sentinel 夹在“项目方 Agent”和“链上执行”之间：先截停拟执行的 RWA 动作，再把它路由到已审核证据、三个 reviewer Agent、人工钱包关口，最后只释放一个有边界的下游 Agent handoff。

hold上游 Agent 的动作先被扣住，什么都不执行。
review结论必须落到具体来源，分歧不能被抹平。
handoff人签字后，下游 Agent 只能做被批准的准备工作。

01 · 问题定义

AI 时代真正危险的不是回答错，而是错了还继续执行。

RWA 项目会把发行、转让、策略准备交给 Agent。Agent 很会补全路径，也很会把“看起来合理”的片段串成一个行动计划。问题是，跨境监管经常不是一个确定答案，而是一组来源边界、分类问题和人工判断。

如果上游 Agent 把“美国看起来有路径”理解成“现在可以推进链上动作”，风险就不再停留在聊天窗口里。

核心问题

需要被管住的不是人问问题，而是 Agent 在证据不够时仍然想继续动手。

02 · 产品定义

Sentinel 是 Agent 和链上执行之间的一道证据闸门。

LUO Sentinel 不给“可以发行”的法律结论。它做的是更窄、更可验证的事：把拟执行动作扣下，展示这个动作目前只能落到哪些已审核来源上，再让 reviewer Agent 检查范围、来源和风险。

只有人用钱包确认之后，系统才生成一个 receipt 和 handoff。这个 handoff 不是“授权发行”，而是“允许下游 Agent 在这个边界内准备给律师看的材料”。

03 · Guarded handoff chain

整条链不是人问 AI，而是 Agent 被 Sentinel 拦下。

01 upstream

项目方 Agent 拟推进 RWA 动作

例如准备 OUSG-like tokenized treasury 在 Injective 上发行或转让。

02 sentinel

Sentinel 截停

动作保持 held 状态，不发交易，不移动资产，不生成结论。

03 evidence

路由到已审核来源

只显示 US、HK、SG、EU 四个已审核信号，不覆盖未审核辖区。

04 human gate

人工钱包关口

人决定 Hold 或 Proceed。Proceed 也只是创建有边界回执。

05 downstream

受限下游 Agent

下游 Agent 只能生成律师准备清单，不能执行发行、转让或策略。

05 · Reviewer Agents

三个 reviewer Agent 不“投票批准”，只负责把风险说清楚。

scope agent

范围是否被偷换

检查请求是否超出已审核辖区和来源范围。

70
source agent

来源是否真能支撑

检查每个 claim 是否有可点击来源，而不是模型自己补出来。

72
risk agent

行动风险是否过高

检查是否出现发行、转让、策略执行等必须由人和律师决定的动作。

68

设计边界

评分不是模型置信度，也不是合规分数；它是给人看的审核权重。

06 · Receipt protocol

链上只记录“谁接受了哪个边界”，不把法律文本塞进链上。

receiptHash: 0x...
evidenceHash: reviewed OUSG pack
productRefHash: OUSG-like treasury sample
reviewerWallet: 0x...
decision: proceed allowedDownstreamAction: counsel-prep checklist
forbidden: issuance, transfer, order, strategy, asset movement
network: Injective EVM testnet
value: 0 INJ

这个 receipt 证明的是：人类审核者在特定 evidence scope 下接受了一个受限 AI handoff。它不证明合规，不构成法律意见，也不授权资产交易。

07 · Demo path

Demo 只证明一件事：Agent 的下一步被约束住了。

主线 · 跨境 review

  1. 01Agent request 被 Sentinel 扣下。
  2. 02地图只打开四个 reviewed source signals。
  3. 03三个 reviewer Agent 给出范围、来源、风险分歧。
  4. 04人选择 Proceed 并用测试网钱包签字。
  5. 05下游 Agent 生成 counsel-prep checklist。

拒绝分支 · 未审核范围

  1. 01Agent 请求超出 reviewed source pack。
  2. 02Sentinel 不编地图，不猜辖区，不给“看起来能做”的答案。
  3. 03用户只能回到跨境范围或单一已审核来源。
  4. 04receipt 仍绑定被选择的 source scope。
  5. 05handoff 不允许扩大到未审核地区。
