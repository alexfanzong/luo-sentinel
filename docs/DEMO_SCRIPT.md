# LUO Sentinel — Three-Minute Demo Script

## 0:00–0:20 — The failure

An AI agent proposes a tokenized-asset action. A fluent answer is not enough:
different jurisdictions can signal different constraints, and a product must
not turn uncertainty into invented certainty.

## 0:20–1:10 — The map

Open the Atlas Canvas. Click the United States, Hong Kong, Singapore, and
European Union to show that each RWA signal remains separately anchored.
Explain that `Cross-border` names the decision context, not a jurisdiction.

## 1:10–1:45 — The human gate

Show the Decision Rail. The product offers no legal conclusion and requires a
human to choose whether to hold the action for counsel or prepare a testnet
decision receipt. It does not establish compliance or authorize an asset action.

## 1:45–2:25 — The Injective path

Explain the current safe integration boundary: a receipt draft contains only a
public action commitment. A user-owned test-only wallet can sign a
non-value-moving Injective testnet receipt only after the app shows the target,
fee estimate, and zero transfer; after deployment, the app re-checks the
deployed runtime code before using it. No key enters the app.

## 2:25–3:00 — The expansion

LUO Sentinel begins with tokenized treasury distribution and expands to RWA
issuance, stablecoins, custody, and cross-border payments. The product value is
not a universal answer; it is a safer next decision.
