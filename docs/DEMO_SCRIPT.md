# LUO Sentinel — Three-Minute Demo Script

## 0:00–0:20 — The starting point

An AI agent proposes a tokenized-asset action. A fluent answer is not enough:
different jurisdictions can signal different constraints, and a product must
not turn uncertainty into invented certainty.

## 0:20–0:45 — The scope

Start from the Agent request capsule. Ask for an action outside the reviewed
source pack and show that Sentinel refuses to fabricate a map. Then choose the
reviewed cross-border OUSG scope, or choose Hong Kong as a single-jurisdiction
path. The first product proof is restraint: an Agent can act only inside a
verified evidence boundary.

## 0:45–1:15 — The map

Open the Atlas Canvas. Click the United States, Hong Kong, Singapore, and
European Union to show that each RWA signal remains separately anchored.
Explain that the selected scope can be comparative or single-jurisdiction, and
that each marker is a source signal, not a market approval.

## 1:15–1:50 — The Review Council

Open the Agent Review Council. Show that the same evidence pack is reviewed by
three deterministic demo agents: Scope Agent, Source Agent, and Risk Agent. In
cross-border mode they check jurisdiction coverage; in Hong Kong-only mode they
shift to source correctness, authority coverage, and applicability risk. This
is what makes LUO more than a database or RAG search. Point out that the scores
come from a fixed demo rubric: coverage, authority fit, claim support, and
residual risk.

## 1:50–2:15 — The human gate

Show the Decision Rail. The product offers no legal conclusion and asks a
human to choose whether to send the action to counsel or prepare a testnet
decision receipt. The action plan stays source-bounded: what to prepare, what
materials are missing, and what must go to local counsel.

## 2:15–2:45 — The Injective path

Explain the current safe integration boundary: a receipt draft contains only a
public action commitment. A user-owned test-only wallet can sign a
non-value-moving Injective testnet receipt only after the app shows the target,
fee estimate, and zero transfer; after deployment, the app re-checks the
deployed runtime code before using it. No key enters the app. Content stays
off-chain; responsibility is anchored on-chain by the receipt hash.

## 2:45–3:00 — The handoff

Show the downstream agent handoff, then click `Run bounded downstream agent`.
The output is a counsel-preparation checklist generated only from the
human-approved scope, reviewer objections, evidence hash, and receipt hash. LUO
Sentinel begins with tokenized treasury distribution and expands to RWA
issuance, stablecoins, custody, and cross-border payments. The product value is
not a universal answer; it is a safer next action boundary.
